# F 0.3.1 — Widget Styling, Style Presets & Listen Variables

## Overview

Extends the host-native `~/.config/wixels/widgets.toml` system with per-widget visual
styling (rounded corners, outer border, concentric inset inner border, padding,
background, typography), reusable `[style.<name>]` presets with per-widget top-level
override keys, symbolic palette color references resolved live at render time, and
`kind = "listen"` variables (the eww `deflisten` equivalent: one long-lived process,
each stdout line becomes the value).

## Problem Statement

`CommandTextView` hardcoded its entire look (`.body` font, padding 14, bg opacity
0.92, radius 12, max width 260); the Cynaberii card aesthetic — outer border plus an
inset second border — was unreachable from TOML. Variables were poll-only, so
streaming sources required wasteful polling.

## Solution Architecture

```
widgets.toml
  [style.card]              → [String: WidgetStylePatch] presets
  [[variable]] kind="listen"→ VariableDefinition (.poll(interval) | .listen)
  [[widget]] style="card"   → WidgetStyle.default
                               .applying(preset)          (unknown name: log + defaults)
                               .applying(widget row keys) (top-level overrides)
                            → TextWidgetDefinition {id, text, placement, style}

Render: CommandTextView resolves each ColorRef against @ObservedObject PaletteStore
inside body — pywal swaps restyle live, no reload, no ThemeContext.

Listen: one supervisor task per listen variable — spawn (shared perl setpgrp
launcher) → read stdout line-by-line → value; on exit, backoff 1s→2s→…→60s
(reset after a ≥30s healthy run), log once per failure streak. Cancellation kills
the process group synchronously; reap escalates SIGTERM → 2s → SIGKILL.

Quit safety: ProcessGroupRegistry tracks live pgids; SIGTERM/SIGINT are routed
through NSApp.terminate and applicationWillTerminate kills every registered group.
```

Note: the originally sketched `style = "card"` + `[widget.style]` override table is
invalid TOML (key redefinition — the whole file would fail to parse). Override keys
therefore sit directly on the widget row; they do not collide with `id`, `text`,
`style`, or placement keys, and presets and overrides share one patch parser.

## Implementation Details

### 1. WidgetStyle

**File**: `Sources/wixels/WidgetStyle.swift` (new)

- `ColorRef`: `.background | .foreground | .accent(Int) | .fixed(RGB)`;
  parses `"background"`, `"foreground"`, `"color0"`–`"color15"`, `"#rrggbb"`
  (via `RGB.init?(hex:)`); resolves against a `Palette` at render time.
- `BorderStyle {width, color}` / `InnerBorderStyle {width, color, inset}` from inline
  tables `{ width = 2, color = "color4", inset = 3 }` (width default 1, color
  foreground, inset 3).
- `WidgetStyle`: Equatable + Sendable; defaults reproduce the pre-style look exactly
  (`fontSize` nil maps to `.body`). `font` is computed, never stored.
- `WidgetStylePatch.parse(from:context:)`: all-optional, **per-key skip-and-log** — a
  bad value drops only that key. Keys: `radius`, `border`, `inner-border`, `padding`,
  `bg`, `bg-opacity`, `max-width`, `align`, `fg`, `font-size`, `font-weight`, `font`
  (`"system"`/`"mono"`/custom name), `line-spacing`, `text-align`.

### 2. Config parsing

**File**: `Sources/wixels/WidgetsConfig.swift`

**Modifications**: `TimedCommand` became `VariableDefinition` with
`VariableKind = .poll(interval:) | .listen`; unknown `kind` skips the variable,
`listen` logs if a meaningless `interval` is present. `[style.*]` presets parse
before the widget loop; each widget resolves default → preset → row patch. A table
`style` value logs a hint to use a preset name plus row keys.

### 3. Store, listen loop, rendering

**File**: `Sources/wixels/CommandTextWidget.swift`

**Modifications**: process construction extracted into `spawn` (perl `setpgrp`
launcher; registers the pgid in `ProcessGroupRegistry`); `run` keeps its 5s
timeout / 8 KiB / exit-status semantics. `listenLoop`/`runListen` implement the
supervisor with per-line delivery (`lines(_:)`, 8 KiB per-line cap), failure-streak
logging via the existing `failing` set, and `withTaskCancellationHandler` whose
`onCancel` is a sync `kill(-pgid, SIGTERM)` (no Swift Concurrency in handlers).
`reap` unregisters the pgid. `CommandTextView` renders from `WidgetStyle`: outer
border as an inside-stroke overlay (fit-content windows never clip it), inner border
as a second overlay `strokeBorder` padded by `inset` with radius clamped to
`max(0, radius - inset)` — the NowPlaying concentric-ring pattern.

### 4. Host quit path

**File**: `Sources/wixels/main.swift`

**Modifications**: `installSignalHandlers()` ignores SIGTERM/SIGINT and re-routes
them through `DispatchSourceSignal` → `NSApp.terminate(nil)`;
`applicationWillTerminate` calls `ProcessGroupRegistry.shared.killAll()`. Without
this, `pkill wixels` orphaned listen children (observed during verification).

## Technical Considerations

- `stop()` stays a synchronous cancel; cancellation kills groups immediately and
  reaping finishes detached. Never await reaping on reload — it would block the main
  actor. Old/new session overlap of ≤2s on reload is by design.
- Overlays add no layout size, so `.fitContent` sizing is unaffected. If
  `padding < inset + width` the text overlaps the inner stroke; intentional, not
  clamped.
- Instant-exit listen commands cannot spin: backoff floor 1s, cap 60s, one log per
  streak.
- Style types stay in the host — no WixelsKit ABI change.

## Backward Compatibility

Existing `widgets.toml` files parse unchanged: no style keys → `WidgetStyle.default`,
pixel-identical to the previous hardcoded card (unset typography maps to `.body`).
`kind` absent → poll with the previous interval semantics.

## Performance & Cost Impact

Style resolution is parse-time; rendering adds two conditional overlays. Each listen
variable costs one child process and one task; line handling is O(line). Backoff
bounds respawn churn.

## Test Impact

`--widgets-config-tests` covers: poll/listen/unknown-kind parsing, preset parsing
from inline tables, preset + row-key override merge, unknown preset fallback,
per-key invalid-value drops, the `ColorRef` parse matrix, and palette resolution.
Verified live: listen clock ticking, group kill on timeout, backoff restart +
recovery logs, reload replacing the process group, zero leaked children after
SIGTERM quit.
